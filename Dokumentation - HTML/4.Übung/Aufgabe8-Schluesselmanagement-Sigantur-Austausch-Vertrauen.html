Web of Trust / Schlüsselmanagement

  • Ein "Web of Trust" (kurz: WOT) wird für das Schlüsselmanagement, zur Weiterleitung von Schlüsseln und deren Zertifzierung verwendet.
    Wenn in einem WOT einem Benutzer den Schlüssel eines anderes Benutzers anvertraut, dann vertraut er auch allen weiteren Schlüsseln, welche von diesem Benutzer signiert wurden.

  • Zunächst werden drei Benutzer erstellt, damit ein WOT simuliert werden kann.
  • Unter Windows wird dazu der Befehl net user /add <benutzername> verwendet.


  • Folgend wird für jeden Benutzer der Befehl runas /user:<benutzername> gpa gpa (eine Graphische Öberfläche für gpg) ausgeführt. Ferner wird für jeden Benutzer ein Schlüsselpaar erstellt.
  • So ergibt sich nun folgende Situation:


  • Als nächsen Schritt wird der Public-Key(PK) des Bestätigenden exportiert und dieser beim Benutzer importiert. Der Schlüssel soll absolutes Vertrauen geniessen.
  • Nun wird der PK vom Dummy exportiert und beim Bestätigenden importiert. Auch hier soll der Schlüssel absolutes Vertrauen geniessen.
    Nun wird der PK vom Dummy vom Bestätigenden signiert.
  • Abschließend wird der signierte PK vom Dummy vom Bestätigenden exportiertund beim Benutzer importiert, der diesen Schlüssel als gültig akzeptiert.
