Add-Ons: UBlock Origin, NoScript, HTTPS Everywhere, Privacy Badger installieren. Fuer Erklaerungen siehe Aufgabe 3.

Einstellungen:
Allgemein: Tabs erst laden, wenn sie ausgewaehlt werden auf true setzen
Suche: DuckDuckGo auswaehlen. DuckDuckGo verfolgt seine User bei der Suche, anders als z.B. Google, nicht.
Inhalt: Pop-Ups blockieren auf true setzen
Datenschutz: Unter "Tracking": Websiten auffordern, meine Aktivitäten nicht zu verfolgen auf true setzen. Dadurch wird an Websiten eine "do-not-track"-Aufforderung gesendet. Allerdings ist es den betroffenen Websiten selbst ueberlassen, dieser Aufforderung nachzukommen. Im selben Abschnitt die Checkbox "Tracking Protection in privaten Fenstern benutzen" auf true setzen. Erklaerung weiter unten unter "about:config".
Chronik auf benutzerdefiniert stellen, "Cookies nur von besuchten Drittanbietern akzeptieren" aktivieren. Ebenfalls koennen Cookies und Verlauf beim Beenden des Firefox geloescht werden.
Sicherheit: Warnen, wenn Websiten versuchen Addons zu installieren auf true setzen. Ebenfalls "Webseite blockieren, wenn sie als attackierend gemeldet wurden" und "Webseite blockieren, wenn sie als Betrugsversuch gemeldet wurde". ""Zugansdaten für Websites speichern" und "Master-Passwort verwenden"nicht aktivieren.
Erweitert:
Allgemein: "Warnen, wenn Webseiten versuchen, umzuleiten oder neuzuladen" auf true setzen.
Datenuebermittlung: Firefox-Absturzbericht und Absturzmelder auf false setzen.
Netzwerk: "Nachfragen, wenn Websiten Daten fuer die Verwendung im Offline-Modus speichern moechten" auf true setzen.

about:config:
trackingprotection.enabled => true. Hierdurch wird die eingebaute Tracking-Protection aktiviert, welche Domains blockt, die den User ueber mehrere Webseiten verfolgen.
geo.enabled => false. Hierdurch koennen Webseiten nicht auf die physische Position des Users, die u.U. auf Laptops gespeichert wird, zugreifen.
network.prefetch-next => false. Erklaerung siehe Aufgabe 1 Teilaufgabe j
network.http.pipelining => true
network.http.pipelining.ssl => true
network.http.proxy.pipelining => true. Durch die drei vorangehenden Einstellungen wird http pipelining aktiviert, was dem Browser erlaubt, mehrere Objekte vom Server parallel zu laden, bspw. Bilder und Text gleichzeitig. Dies kann zwar bei alten oder schlecht konfigurierten Webseiten Probleme bereiten, stellt aber bei den meisten Webseiten eine Geschwindigkeitsverbesserung dar.
loop.enabled => false
browser.pocket.enable => false
media.navigator.enabled => false
media.peerconnection.enabled => false. Die drei vorhergehenden Einstellungen deaktivieren die Firefox-Applikationen Pocket, Hello und WebRTC. Diese werden von den meisten Usern nicht benoetigt und sollten daher deaktiviert werden.
