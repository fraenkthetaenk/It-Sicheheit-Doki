Intrusion Detection / AIDE

a)
AIDE legt bei der ersten Ausführung eine Datenbank an, in welche der aktuelle Stand der zu sichernden Dateien (hier: /etc) abgebildet wird. Unter anderem werden hier Informationen wie Berechtigungen, Zeitpunkt der letzten Änderung und/oder der Inhalt der Dateien festgelegt.
Mit einem Kommando (siehe unten) vergleicht AIDE die zu überwachenden Daten mit der angelegten "Wunsch"-Datenbank. Dabei vergleicht es die Prüfsumme der initialen Datenbank mit der überwachten Datei.

b) Die Installation geschieht mit

c)
Zuerst muss eine Initial-DB angelegt werden.
Dies kann durch
sudo aide -i
passieren.
Eine DB kann mit
sudo aide -c /etc/aide/aide.conf --init
erzeugt werden. Allerdings wird hier in /var/lib/aide eine aude.db.new erstellt, welche mit
sudo mv /var/lib/aide/aide.db.new /var/lib/aide/aide.db
zur "richtigen" DB wird. Die muss jedesmal getan werden, wenn durch aide --update ein neues Abbild der Verzeichnisse angelegt wird.
In /etc/aide/aide.conf können unter den DB-Pfaden die AIDE-Regeln definiert werden. In diesem Fallbeispiel soll nur /etc überprüft werden.
